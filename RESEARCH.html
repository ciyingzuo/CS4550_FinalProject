Option 1: Player stats tracker

Problem statement:

Our goal is to make a web application that can let player view their game stats easily. The functionality will include match history, kill and death statics, accuracy, weapon/gear stats. In addition we will calculate a overall score and put them on global leader board so people can compare with other players

Target audience:

Game player, development team.

Solution proposal:

We will use a server to retrieve the data first, then we will process and store it on our own server. When people search their player ID on web page, we will send the information to our frontend and render the information in various ways, eg chart.

Data source:

Data will be retrieve from Trinet. It's public available.

Option 2: Online real time chatting software

Problem statement:

In this project we will make a online chat application. Anyone can register as a member. People can do online chat in real time, they can also add other member as friends, or join a group to chat we multiple user at the same time. Each user will have their own personal page, where they can upload photos and share it. Their friend can view the page and make comment.

Target audience:

Everyone.

Solution proposal:

we will host a database and a server as our backend, database will store user information. The server will communicate with frontend and response for a series action like login register send message. The front will receive the information from the server and render it.

Data source:

we will use our own database, all data is from user action: register, post message.
